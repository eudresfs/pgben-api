# Sistema de Gestão de Benefícios Eventuais - SEMTAS

## Visão Geral
Sistema digital para automatizar e gerenciar todo o ciclo de vida das solicitações de benefícios eventuais da SEMTAS de Natal/RN, conforme Lei Municipal 7.205/2021 e Decreto Municipal 12.346/2021.

## Objetivos
- Digitalizar completamente o processo de solicitação e concessão de benefícios
- Implementar controle rigoroso de exclusividade de papéis (beneficiário vs. composição familiar)
- Reduzir tempo de análise e aprovação das solicitações
- Melhorar rastreabilidade e auditoria do processo
- Garantir conformidade com LGPD
- Facilitar acesso aos serviços assistenciais
- Gerar relatórios e indicadores de gestão
- Suportar determinações judiciais

## MVP - Benefícios Iniciais
- Benefício Natalidade (kit enxoval)
- Aluguel Social

## Funcionalidades Principais

### 1. Gestão de Usuários e Autenticação
- Sistema de login com JWT
- Controle de acesso baseado em perfis (RBAC)
- Gestão de usuários por unidades e setores
- Middleware de rate limiting e segurança
- Validação de força de senha

### 2. Cadastro de Cidadãos
- Cadastro completo de cidadãos
- Validação de CPF e documentos
- Gestão de composição familiar
- Controle de exclusividade (beneficiário não pode ser composição familiar)

### 3. Gestão de Solicitações
- Criação de solicitações de benefícios
- Upload e gestão de documentos
- Workflow de aprovação (análise técnica → aprovação → liberação)
- Controle de status e histórico
- Notificações automáticas

### 4. Gestão de Benefícios
- Configuração de tipos de benefícios
- Campos dinâmicos por tipo de benefício
- Regras de elegibilidade
- Controle de periodicidade e limites

### 5. Gestão Documental
- Upload seguro de documentos
- Criptografia de arquivos sensíveis
- Controle de acesso a documentos
- Auditoria de visualizações

### 6. Auditoria e Logs
- Log completo de todas as operações
- Rastreabilidade de mudanças
- Relatórios de auditoria
- Conformidade com LGPD

### 7. Relatórios e Métricas
- Dashboard executivo
- Relatórios de benefícios concedidos
- Métricas de performance
- Indicadores de gestão

### 8. Determinações Judiciais
- Cadastro de determinações judiciais
- Processamento extraordinário
- Controle de cumprimento
- Relatórios específicos

## Requisitos Técnicos

### Backend
- Node.js 20 LTS
- NestJS framework
- TypeScript
- PostgreSQL
- TypeORM
- JWT para autenticação
- Swagger para documentação

### Segurança
- Criptografia de dados sensíveis
- Headers de segurança HTTP
- Rate limiting
- Validação robusta de inputs
- Conformidade com LGPD

### Infraestrutura
- Azure App Service
- Azure Database for PostgreSQL
- Azure Blob Storage
- Docker containers
- CI/CD com GitHub Actions

## Critérios de Aceitação
- Sistema deve processar solicitações de benefícios de forma digital
- Controle rigoroso de perfis e permissões
- Auditoria completa de todas as operações
- Interface intuitiva para usuários
- Performance adequada (< 2s para operações principais)
- Disponibilidade de 99.5%
- Conformidade total com LGPD

## Restrições
- Prazo de 1 mês para MVP
- Orçamento limitado
- Equipe reduzida
- Conformidade obrigatória com legislação municipal
- Integração futura com sistemas existentes da prefeitura
